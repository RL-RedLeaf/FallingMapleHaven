### P1-3: User 与 Profile 模型职责边界不清晰

| 文档 | 问题 |
|------|------|
| `02-技术架构#2.2.1` | accounts_user 表包含 nickname, avatar, cover_image, bio, real_name |
| `02-技术架构#2.2.2` | profiles_profile 表包含 show_real_name, visitor_public, layout_config |

**分析**: User 存显示数据，Profile 存配置数据。但 API `GET /api/v1/auth/me/` 返回的数据中同时包含两者字段。字段归属不清晰会造成开发时的混淆。

**建议**: 明确以下约定：
- **User 模型** (`django.contrib.auth.User` 扩展): nickname, avatar, cover_image, bio, real_name, email
- **Profile 模型** (OneToOneField → User): show_real_name, visitor_public, layout_config
- 前端通过 `auth/me/` 获取合并数据，后端在 Serializer 中做展平
- 修改 `02-技术架构#2.2.1` 表说明，标注哪些字段是通过 `AbstractUser` 扩展，哪些在 Profile

### P2-1: 插件 API 路径风格不一致

| 插件 | API 路径 |
|------|----------|
| 默契问答 | `POST /api/v1/plugins/quiz/questions/` |
| 匿名提问箱 | `POST /api/v1/plugins/questions/` |

**问题**: 匿名提问箱的路径比默契问答少一层 `question_box/` 前缀，风格不一致。且命名 `questions` 与默契问答的 `questions` 容易混淆。

**建议**: 统一为：
```
默契问答:     /api/v1/plugins/quiz/questions/
匿名提问箱:   /api/v1/plugins/question-box/questions/
```

| 涉及文档 | 问题 |
|---------|------|
| `03-API与WebSocket接口文档.md` | 没有任何留言板 API 定义 |
| `04-前端#2.1` | 存在路由 `/profile/:userId/guestbook` |
