# 开发协作规范 — FallingMapleHaven

---

## 1. Git 工作流

### 1.1 分支策略

```
main          ── 稳定可部署版本
dev           ── 日常开发集成分支
feat/<name>   ── 功能分支 (从 dev 切出)
fix/<name>    ── 修复分支 (从 dev 切出)
```

### 1.2 提交信息规范

```
<type>(<scope>): <description>

类型: feat / fix / refactor / style / docs / chore / test
示例:
  feat(auth): 实现用户注册 API
  fix(chat): 修复 WebSocket 断线重连
  refactor(feed): 提取 PostCard 公共组件
  docs: 更新 API 文档
  chore: 更新依赖版本
```

### 1.3 单人开发简化流程

```bash
# 日常开发
git checkout dev
git pull
git checkout -b feat/xxx    # 创建功能分支
# ... 开发 ...
git add .
git commit -m "feat(xxx): ..."
git checkout dev
git merge feat/xxx
git branch -d feat/xxx
git push origin dev
```

单人开发时 dev 即为日常主线，功能分支用于隔离较大改动。

## 2. 后端编码规范

### 2.1 目录约定

```
apps/<app_name>/
├── models.py         # 数据模型
├── serializers.py    # DRF 序列化器
├── views.py          # API 视图 (ViewSet)
├── urls.py           # 路由 (router.register)
├── filters.py        # 过滤器 (可选)
├── permissions.py    # 权限类 (可选)
└── signals.py        # 信号 (可选)
```

### 2.2 代码风格

- **缩进**: 4 空格 (无 Tab)
- **命名**: `snake_case` 变量/函数/文件, `PascalCase` 类
- **import 顺序**: 标准库 → Django → DRF → 项目内部
- **行长度**: 88 字符 (Black 默认)
- **类型注解**: 函数参数和返回值需标注类型

### 2.3 DRF 最佳实践

```python
# ✅ 使用 ViewSet + Router
class PostViewSet(ModelViewSet):
    queryset = Post.objects.all()
    serializer_class = PostSerializer
    permission_classes = [IsAuthenticated]

    def perform_create(self, serializer):
        serializer.save(author=self.request.user)

# ❌ 避免使用 Function Based View
# ❌ 避免在 views.py 写复杂业务逻辑 — 使用 serializers 或 services

# ✅ 统一响应格式 (使用自定义渲染器)
# { "code": 0, "message": "success", "data": {...} }
```

### 2.4 安全规范

```python
# 密码
from django.contrib.auth.hashers import make_password
user.password = make_password(raw_password)

# 文件上传校验
def validate_image(file):
    import imghdr
    if imghdr.what(file) not in ['jpeg', 'png', 'gif', 'webp']:
        raise ValidationError("不支持的图片格式")

# SQL 注入防护 — 始终使用 ORM
# ✅ Post.objects.filter(author=user)
# ❌ 不要使用 raw() 或 extra() 拼接用户输入

# XSS 防护
# Django 模板自动转义; DRF 序列化输出 JSON 无此问题
```

### 2.5 数据库操作

```python
# 大量创建使用 bulk_create
PostImage.objects.bulk_create(images)

# 查询优化使用 select_related / prefetch_related
post = Post.objects.select_related('author').get(id=1)

# 更新使用 update_fields
post.save(update_fields=['content'])
```

## 3. 前端编码规范

### 3.1 命名约定

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件名 | PascalCase, 多词 | `PostCard.vue`, `MessageBubble.vue` |
| 路由路径 | kebab-case | `/profile/:userId` |
| API 函数 | camelCase | `fetchPosts()`, `sendMessage()` |
| Pinia Store | camelCase | `useAuthStore`, `useChatStore` |
| CSS 类 | Tailwind utility | 避免自定义 CSS |

### 3.2 组件规范

```vue
<script setup>
// 1. imports
import { ref, computed } from 'vue'
import { useAuthStore } from '@/stores/auth'

// 2. props (类型声明)
const props = defineProps({
  post: { type: Object, required: true },
  showActions: { type: Boolean, default: true },
})

// 3. emits
const emit = defineEmits(['like', 'comment'])

// 4. reactive state
const loading = ref(false)

// 5. computed
const isOwner = computed(() => props.post.author.id === authStore.userId)

// 6. methods
async function handleLike() {
  emit('like', props.post.id)
}
</script>

<template>
  <!-- 模板保持简洁，复杂逻辑移至 script -->
</template>
```

### 3.3 API 层规范

```javascript
// src/api/client.js
import axios from 'axios'

const client = axios.create({
  baseURL: '/api/v1',
  withCredentials: true,
  headers: { 'Content-Type': 'application/json' },
})

client.interceptors.response.use(
  (response) => response.data,  // 自动解包 { code, message, data }
  (error) => {
    const msg = error.response?.data?.message || '网络错误'
    // 全局错误 Toast
    return Promise.reject(new Error(msg))
  }
)

export default client

// src/api/auth.js
import client from './client'

export const authApi = {
  register: (data) => client.post('/auth/register/', data),
  login: (data) => client.post('/auth/login/', data),
  logout: () => client.post('/auth/logout/'),
  me: () => client.get('/auth/me/'),
  updateMe: (data) => client.patch('/auth/me/', data),
  changePassword: (data) => client.post('/auth/change-password/', data),
  uploadAvatar: (file) => client.post('/auth/avatar/', { avatar: file }, {
    headers: { 'Content-Type': 'multipart/form-data' },
  }),
}
```

### 3.4 Store 规范

```javascript
// src/stores/auth.js
import { defineStore } from 'pinia'
import { authApi } from '@/api/auth'

export const useAuthStore = defineStore('auth', {
  state: () => ({
    user: null,
    loading: false,
  }),

  getters: {
    isAuthenticated: (state) => !!state.user,
    isAdmin: (state) => state.user?.is_admin || false,
  },

  actions: {
    async login(username, password) {
      this.loading = true
      try {
        const res = await authApi.login({ username, password })
        this.user = res.data
      } finally {
        this.loading = false
      }
    },
    async fetchMe() {
      const res = await authApi.me()
      this.user = res.data
    },
    async logout() {
      await authApi.logout()
      this.user = null
    },
  },
})
```

## 4. 接口联调规范

### 4.1 前后端约定

- 后端先完成 API → 提供 Swagger/文档 → 前端对接
- 响应格式统一：`{ code: 0, message: "success", data: {...} }`
- 分页格式统一：`{ count, next, previous, results }` (DRF 默认)
- 错误码见 `03-API#1.2`

### 4.2 开发代理

Vite 配置代理转发 `/api` → Django `:8000`，避免跨域问题。
前端开发时无需启动 Nginx。

### 4.3 调试技巧

```bash
# 后端打印 SQL
python manage.py runserver --verbosity=3

# 后端 Shell 调试
python manage.py shell

# 前端 Vue DevTools
# 安装 Chrome 扩展: Vue.js devtools
```

## 5. 测试规范

### 5.1 后端测试

```bash
# 运行所有测试
python manage.py test

# 运行单个 App 测试
python manage.py test apps.accounts

# 测试覆盖率
pip install coverage
coverage run --source='apps' manage.py test
coverage report
```

### 5.2 测试编写要求

- 每个 API 至少一个成功用例 + 一个异常用例
- 使用 Django `TestCase` 或 DRF `APITestCase`
- 使用 `factory_boy` 创建测试数据

### 5.3 前端测试

```bash
npm run test:unit    # Vitest
npm run test:e2e     # Playwright (可选)
```
