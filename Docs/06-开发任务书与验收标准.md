## Phase 1: 地基搭建

### 任务 1.1 — Django 项目初始化

**开发人:** 后端  
**估时:** 2 小时  
**依赖:** 无  

**具体内容:**
1. 使用 Django 5.x 创建项目 `config/`
2. 配置 `settings/dev.py` 和 `settings/prod.py`
3. 配置 PostgreSQL 数据库连接
4. 配置 `.env` 管理密钥（使用 `django-environ`）
5. 配置 `CORS` 和 `CSRF` 信任源
6. 创建 Django App: `accounts`, `profiles`, `friends`, `feed`, `chat`, `groups`, `plugins`, `notifications`, `admin_dashboard`
7. 配置 `ASGI` 入口（集成 Channels）
8. 安装并配置 `daphne` 作为 ASGI 服务器
9. 配置 `redis` 作为 Channel Layer 后端

**交付物:**
- [ ] Django 项目可启动
- [ ] 数据库表可创建
- [ ] Redis 连接正常
- [ ] ASGI 服务可启动

### 任务 1.3 — 用户系统（后端）

**开发人:** 后端  
**估时:** 4 小时  
**依赖:** 1.1  

**具体内容:**
1. 扩展 Django `AbstractUser`，添加 `nickname`, `real_name`, `avatar`, `cover_image`, `bio` 字段
2. 实现注册 API: `POST /api/v1/auth/register/`
   - 校验 username 唯一性
   - 密码强度（>=8位，含字母+数字）
   - 自动创建 Profile
3. 实现登录 API: `POST /api/v1/auth/login/`（返回用户信息）
4. 实现退出 API: `POST /api/v1/auth/logout/`
5. 实现获取当前用户: `GET /api/v1/auth/me/`
6. 实现更新资料: `PATCH /api/v1/auth/me/`
7. 实现修改密码: `POST /api/v1/auth/change-password/`
8. 实现头像上传: `POST /api/v1/auth/avatar/`
9. 实现封面上传: `POST /api/v1/auth/cover/`

**验收标准:**
- [ ] 使用用户名+密码注册成功
- [ ] 重复用户名返回错误码 40001
- [ ] 登录成功返回用户信息
- [ ] 未登录访问需认证接口返回 401
- [ ] 头像上传成功可访问
- [ ] 密码修改后旧密码失效
