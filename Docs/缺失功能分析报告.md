# 当前版本缺口分析报告

> 日期：2026-06-28  
> 范围：按原始 PRD/设计文档的当前版本边界，对比实际代码实现。  
> 结论：此前 16 项缺口中，多数已由代码补齐或属于当前版本暂不考虑范围；本轮已补齐管理后台前端、匿名追溯入口、评论管理和插件禁用一致性。

## 一、版本边界

当前版本按 PRD 的 P0/P1 功能验收，主要包括：

- 用户注册/登录、个人资料、访客足迹
- 好友系统
- 社交广场、动态、点赞、评论、话题筛选
- 私聊/群聊文字聊天
- 个人主页、留言板
- 通知系统
- 首批互动插件：默契问答、匿名提问箱
- 兴趣小组
- 管理后台

以下内容不计入当前版本缺口：

- Phase 6 / P2 / P3 功能：纪念日倒计时、许愿池、投票问卷、在线小游戏、音乐分享
- 小组投票、小组相册、小组公告等“扩展预留”
- 聊天图片/文件消息等首批文字聊天之外的能力
- Dockerfile、Nginx、生产部署文档
- Redis `CACHES` 配置（当前保持开发环境优先）
- 大规模组件拆分、插件动态路由、额外 Pinia Store 等架构优化项

## 三、本轮已处理的真实缺口

### C1 — 管理后台前端未接通后端 API（已完成）

**依据：** PRD 将“管理后台”列为 Phase 5 / P0，属于当前版本应收口功能。

**当前代码：**

- 后端已有多组管理 API：用户、动态、插件、站点设置、统计、日志、匿名提问追溯。
- 前端管理页面此前多数仍是静态壳，用户、内容、插件、设置、统计、日志页面均未接通 API。
- `frontend/src/api/` 下没有 `admin.js`，也没有 `index.js` 聚合导出管理 API。

**影响：** 管理后台页面可进入，但不能真实管理数据。

**处理结果：** 已新增 `frontend/src/api/admin.js`，并接通用户、动态、评论、插件、站点设置、统计和日志页面。

---

### C2 — 匿名提问追溯后端存在，前端入口不可达（已完成）

**依据：** PRD 要求管理员能查看匿名提问真实用户，用于追溯处理。

**当前代码：**

- 后端 API 已存在：`GET /api/v1/admin/anonymous-questions/`
- `AdminAnonymousQuestionsPage.vue` 已存在，但：
  - 未挂到 `frontend/src/router/admin.js`
  - 未加入 `AdminDashboard.vue` 侧边栏导航
  - 页面中使用 `import { adminApi } from '@/api'`，当前无对应聚合导出

**影响：** 管理员无法从前端访问匿名提问追溯页面。

**处理结果：** 已挂载 `/admin/anonymous-questions` 路由，加入后台导航，并改为从 `@/api/admin` 导入。

---

### C3 — 管理后台评论管理闭环（已完成）

**依据：** PRD §2.9.2 明确包含评论管理：查看/删除评论。

**当前代码：**

- `admin_dashboard/urls.py` 当前没有 `comments/` 管理路由。
- `AdminPostsPage.vue` 只预留动态列表，未包含评论列表/删除操作。

**影响：** 管理员可以管理动态，但无法按 PRD 管理评论。

**处理结果：** 已增加后端评论列表/删除接口，并在内容管理页加入评论 Tab。

---

### C4 — 插件禁用一致性（已完成）

**依据：** 插件系统设计要求管理员可启停插件；禁用插件后应从用户视角不可用。

**当前代码：**

- `PluginRegistry.get_active_plugins()` 会按 `Plugin.is_active` 过滤主页插件入口。
- 具体插件 API（默契问答、匿名提问箱）未统一检查插件是否启用。

**影响：** 管理员禁用插件后，主页入口会隐藏，但用户如果直接访问已知页面/API，仍可能继续操作。

**处理结果：** 已在默契问答和匿名提问箱 API 层增加启用状态检查，禁用时返回 404。
