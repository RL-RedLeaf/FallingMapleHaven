## 1. 背景与目标

### 1.1 当前问题

管理后台 (`/admin`) 的侧边栏导航和路由全部**硬编码**在 `AdminDashboard.vue` 和 `admin.js` 中。每增加一个新插件的管理页面，都需要手动修改这两处文件，不符合插件系统的可插拔设计目标。

### 1.2 设计目标

- **插件自声明**：每个插件通过标准接口声明自己的管理页面
- **自动发现**：管理后台自动收集所有已注册插件的管理页面配置
- **前后端协同**：后端插件声明管理 API 端点，前端插件声明管理页面路由和侧边栏
- **向后兼容**：现有硬编码管理页面不受影响，逐步迁移
- **零侵入**：新增插件管理页不需要修改 `admin_dashboard` 核心代码

## 3. 后端设计

### 3.1 `get_admin_config()` 接口定义

```python
class BasePlugin(ABC):
    # ... 现有属性 ...

    def get_admin_config(self) -> dict:
        """
        返回插件在管理后台的配置。

        返回格式:
        {
            "sidebar": [                       # 侧边栏菜单项（可选，多个）
                {
                    "path": "questions",       # 相对于 /admin/ 的路径，如 "questions"
                    "name": "提问箱管理",       # 菜单显示名称
                    "icon": "🕵️",              # 菜单图标
                }
            ],
            # 无需声明 URL 路由 —— 插件自己的 urls.py 已挂载到 /api/v1/plugins/ 下
            # admin_dashboard 通过插件类型自动代理到前端路由
        }
        """
        return {}
```

### 3.2 插件实现示例

```python
# backend/apps/plugins/questions/plugin.py

@register_plugin
class QuestionBoxPlugin(BasePlugin):
    plugin_type = "question_box"
    display_name = "匿名提问箱"
    # ... 现有属性 ...

    def get_profile_data(self, user_id):
        # ... 现有实现 ...

    def get_admin_config(self):
        return {
            "sidebar": [
                {
                    "path": "anonymous-questions",
                    "name": "提问箱管理",
                    "icon": "🕵️",
                }
            ]
        }
```

### 3.3 后端自动发现机制

管理后台通过 `PluginRegistry` 自动收集所有插件的 admin 配置：

```python
# admin_dashboard 中新增收集逻辑

from apps.plugins.registry import PluginRegistry

def get_plugin_admin_configs():
    """返回所有插件的管理后台配置"""
    configs = []
    for plugin in PluginRegistry.get_all_plugins().values():
        admin_config = plugin.get_admin_config()
        if admin_config and admin_config.get("sidebar"):
            configs.append({
                "type": plugin.plugin_type,
                "name": plugin.display_name,
                **admin_config,
            })
    return configs
```

**不新增 API 端点**：侧边栏配置通过现有 admin API 的扩展返回（如嵌入 `GET /admin/settings/` 响应，或新增 `GET /admin/plugins/config/`）。

### 3.4 URL 路由映射规则

插件管理页面调用的 API 端点遵循现有模式：

| 类型 | URL 模式 | 示例 |
|------|---------|------|
| 插件业务 API | `/api/v1/plugins/{type}/...` | `GET /api/v1/plugins/question-box/inbox/` |
| 管理后台 API | `/api/v1/admin/...` | `GET /api/v1/admin/anonymous-questions/` |

**过渡期**：现有 `AdminAnonymousQuestionsView` 保留在 `admin_dashboard/views.py` 中，URL 挂载在 `/api/v1/admin/` 下。

**远期**：插件可自行注册 admin API 端点，通过 `urls.py` 中声明，由 `admin_dashboard` 的根 URLConf 自动 include。

## 4. 前端设计

### 4.1 PluginRegistry 扩展

```javascript
// frontend/src/plugins/registry.js

class PluginRegistry {
  constructor() {
    this.plugins = new Map()
  }

  register(pluginConfig) {
    // 扩展: 新增 adminRoute 和 adminSidebar 字段
    // pluginConfig: {
    //   type, name, icon,
    //   route: { path, name },                    // 用户侧路由（现有）
    //   adminRoute: { path, name, component },     // 管理侧路由（新增）
    //   adminSidebar: { name, icon },              // 管理侧边栏（新增）
    // }
    this.plugins.set(pluginConfig.type, pluginConfig)
  }

  // ... 现有方法 ...

  getAdminRoutes() {
    // 新增: 收集所有插件的 adminRoute
    const routes = []
    for (const plugin of this.plugins.values()) {
      if (plugin.adminRoute) {
        routes.push({
          ...plugin.adminRoute,
          meta: { requiresAuth: true, requiresAdmin: true },
        })
      }
    }
    return routes
  }

  getAdminSidebarItems() {
    // 新增: 收集所有插件的 adminSidebar
    const items = []
    for (const plugin of this.plugins.values()) {
      if (plugin.adminSidebar && plugin.adminRoute) {
        items.push({
          path: plugin.adminRoute.path,
          name: plugin.adminSidebar.name,
          icon: plugin.adminSidebar.icon,
        })
      }
    }
    return items
  }
}
```

## 7. 数据契约

### 7.1 后端 `get_admin_config()` 返回值

```json
{
  "sidebar": [
    {
      "path": "anonymous-questions",
      "name": "提问箱管理",
      "icon": "🕵️"
    }
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `sidebar` | `array` | 否 | 侧边栏菜单配置，不提供则不显示侧边栏项 |
| `sidebar[].path` | `string` | 是 | 相对于 `/admin/` 的 Vue Router 路径 |
| `sidebar[].name` | `string` | 是 | 侧边栏显示名称 |
| `sidebar[].icon` | `string` | 是 | 菜单图标（支持 emoji 或 lucide 图标名） |

### 7.2 前端 `pluginConfig` 扩展字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `adminRoute` | `object` | 否 | Vue Router 路由配置 |
| `adminRoute.path` | `string` | 是 | 相对于 `/admin/` 的路径 |
| `adminRoute.name` | `string` | 是 | 路由名称 |
| `adminRoute.component` | `Component` | 是 | 页面组件 |
| `adminSidebar` | `object` | 否 | 侧边栏菜单配置 |
| `adminSidebar.name` | `string` | 是 | 菜单显示名称 |
| `adminSidebar.icon` | `string` | 是 | 菜单图标（支持 emoji 或 lucide 图标名） |

## 8. 注意事项

1. **插件不存在时不应中断**：如果某个插件注册了但被卸载，其 admin 配置应被忽略，不抛异常
2. **侧边栏顺序**：内置项固定在前，插件项在后；插件之间按注册顺序排列
3. **路由名称冲突**：插件路由 name 必须加前缀以避免冲突（如 `AdminQuestions_QuestionBox`）
4. **组件懒加载**：插件 admin 页面组件应使用动态 `import()` 按需加载
5. **权限统一**：所有插件 admin 路由统一继承父路由的 `meta: { requiresAuth: true, requiresAdmin: true }`
6. **新插件添加步骤**（含管理页面）：
   - 后端：创建 `plugin.py` → 实现 `get_admin_config()` → 编写管理 API 视图
   - 前端：创建 admin 页面组件 → 在 `index.js` 中注册 `adminRoute` + `adminSidebar`
