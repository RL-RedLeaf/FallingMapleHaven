## 1. 技术选型

### 1.1 总体架构图

```
┌─────────────────────────────────────────────────┐
│                  浏览器 (Chrome/Safari)           │
│          Vue 3 SPA + Tailwind CSS                │
├──────────────────────┬──────────────────────────┤
│   HTTP (REST API)    │   WebSocket (实时通信)     │
├──────────────────────┴──────────────────────────┤
│              Nginx (反向代理)                     │
├──────────────────────┬──────────────────────────┤
│   Django + DRF       │   Django Channels         │
│   (API 服务)          │   (WebSocket 服务)        │
├──────────────────────┴──────────────────────────┤
│              Redis (缓存 + Channel Layer)         │
├─────────────────────────────────────────────────┤
│              PostgreSQL (主数据库)                 │
├─────────────────────────────────────────────────┤
│              本地文件系统 (媒体存储)                │
└─────────────────────────────────────────────────┘
```

### 1.2 各层技术明细

| 层级 | 技术 | 版本 | 用途 |
|------|------|------|------|
| 后端框架 | Django | 5.x | 核心业务逻辑、ORM、Admin |
| API | Django REST Framework | 3.15+ | RESTful API 接口 |
| 实时通信 | Django Channels | 4.x | WebSocket 聊天、实时通知 |
| ASGI 服务器 | Daphne / Uvicorn | 最新 | 运行 Channels |
| 数据库 | PostgreSQL | 16 | 关系数据存储 |
| 缓存/消息 | Redis | 7.x | Channel Layer、缓存、Session |
| 前端框架 | Vue 3 | 3.x | SPA 前端 |
| 构建工具 | Vite | 5.x | 前端构建 |
| 状态管理 | Pinia | 2.x | 前端全局状态 |
| 路由 | Vue Router | 4.x | 前端路由 |
| UI 框架 | Tailwind CSS | 3.x | 原子化样式 |
| 反向代理 | Nginx | 最新 | 静态文件 + API 代理 |
| 容器化 | Docker + Compose | 最新 | 部署 |

### 1.3 选型理由

| 选择 | 理由 |
|------|------|
| Django | 内置 Admin、Auth、ORM，成熟稳定，减少重复造轮子 |
| DRF | 与 Django 深度集成，ViewSet/Serializer 体系高效 |
| Channels | 原生 Django WebSocket 方案，共享 Django 认证和 ORM |
| Vue 3 | 组件化开发，Composition API 灵活，生态成熟 |
| Tailwind | 快速定制秋叶主题色，响应式断点便捷 |
| PostgreSQL | 功能强大，支持 JSONField 适合灵活配置存储 |

### 2.2 所有表结构定义

#### 2.2.1 accounts_user (Django 内置 User 扩展)

> **字段归属约定**: nickname, real_name, avatar, cover_image, bio 等显示性字段直接放在 User 模型（通过 AbstractUser 扩展）；show_real_name, visitor_public, layout_config 等配置性字段放在 Profile 模型。前端通过 API 获取合并后的展平数据。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT AUTO_INCREMENT | PK | 用户ID |
| username | VARCHAR(150) | UNIQUE, NOT NULL | 登录用户名 |
| password | VARCHAR(128) | NOT NULL | 加密密码 |
| email | VARCHAR(254) | NULL | 邮箱（选填） |
| nickname | VARCHAR(50) | NOT NULL | 显示昵称 |
| real_name | VARCHAR(50) | NULL | 真实姓名 |
| avatar | VARCHAR(200) | NULL | 头像路径 |
| cover_image | VARCHAR(200) | NULL | 封面图路径 |
| bio | TEXT | NULL | 个人签名 |
| is_active | BOOLEAN | DEFAULT TRUE | 是否激活 |
| is_staff | BOOLEAN | DEFAULT FALSE | 是否管理员 |
| is_superuser | BOOLEAN | DEFAULT FALSE | 是否超级管理员 |
| last_login | DATETIME | NULL | 最后登录时间 |
| date_joined | DATETIME | NOT NULL | 注册时间 |

> Django 会自动创建 auth_user 表，通过 `AbstractUser` 扩展上述字段

## 3. 数据迁移顺序

开发时应按以下顺序创建 Django App 和迁移：

```
1. accounts          ──→ 基础用户表
2. profiles          ──→ Profile + VisitRecord + GuestbookEntry
3. friends           ──→ Friendship
4. feed              ──→ Post + PostImage + PostFile + Comment + Like
5. chat              ──→ ChatRoom + ChatRoomMember + Message + MessageRead
6. groups            ──→ InterestGroup + GroupMember
7. plugins           ──→ Plugin + Quiz* + AnonymousQuestion
8. notifications     ──→ Notification
9. admin_dashboard   ──→ SiteLog + SiteSettings
```
