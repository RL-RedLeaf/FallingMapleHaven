# 访客公开浏览功能设计方案

## 一、概述

### 1.1 背景

当前网站全站需要登录才能访问。未登录用户只能看到 `/login` 和 `/register` 两个页面，首页 `/` 重定向到登录页，导航栏对访客完全隐藏。这导致新用户无法在注册前了解社区内容，转化路径不顺畅。

### 1.2 目标

将网站从"全站需登录"改为"访客可浏览公开内容，登录后可交互"的模式，在不影响已有功能的前提下打开流量入口。

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| **访客可读，用户可写** | 公开数据的 GET 请求对访客开放，所有写操作（POST/PUT/PATCH/DELETE）保持身份验证 |
| **渐进增强** | 访客看到的内容是登录用户所见的子集，登录后解锁更多操作能力 |
| **最小变更** | 不改动既有数据模型、URL 结构、序列化器，只调整权限控制和视图层的分支逻辑 |
| **安全优先** | 私有数据（仅好友/私密动态、访客列表、聊天记录等）始终受 IsAuthenticated 保护 |

### 2.2 API 端点级权限

#### 2.2.1 对访客开放的 GET 端点

| 端点 | 方法 | 说明 | 访客返回范围 |
|------|------|------|-------------|
| `/api/v1/posts/` | GET | 动态列表 | 仅 `visibility=public` 的帖子 |
| `/api/v1/posts/{id}/` | GET | 动态详情 | 仅公开帖子 |
| `/api/v1/posts/{id}/comments/` | GET | 评论列表 | 仅公开帖子的评论 |
| `/api/v1/profiles/{username_or_id}/` | GET | 用户资料 | 公开资料（头像/昵称/简介/插件列表） |
| `/api/v1/profiles/{username_or_id}/posts/` | GET | 用户动态 | 仅该用户的公开帖子 |
| `/api/v1/profiles/{username_or_id}/guestbook/` | GET | 留言板 | 全部留言（留言板本身是公开板块） |
| `/api/v1/groups/` | GET | 群组列表 | 仅 `is_public=True` 的群组 |
| `/api/v1/auth/csrf/` | GET | CSRF Token | 完全开放（已有） |

#### 2.2.2 保持登录保护的端点

以下端点不受影响，保持 `permission_classes = [IsAuthenticated]`：

**Feed 模块：**
- `POST /api/v1/posts/create/` — 发布动态
- `DELETE /api/v1/posts/{id}/delete/` — 删除动态
- `POST /api/v1/posts/{id}/like/` — 点赞/取消
- `POST /api/v1/posts/{id}/comments/create/` — 发表评论
- `DELETE /api/v1/posts/comments/{id}/delete/` — 删除评论

**Profiles 模块：**
- `GET /api/v1/profiles/{username_or_id}/visitors/` — 访客列表（隐私保护）
- `POST /api/v1/profiles/{username_or_id}/guestbook/create/` — 写留言
- `POST /api/v1/profiles/guestbook/{entry_id}/reply/` — 回复留言
- `DELETE /api/v1/profiles/guestbook/{entry_id}/delete/` — 删除留言

**Groups 模块：**
- `POST /api/v1/groups/create/` — 创建小组
- `GET /api/v1/groups/{id}/` — 群组详情
- `PATCH /api/v1/groups/{id}/update/` — 更新小组信息
- `POST /api/v1/groups/{id}/join/` — 加入小组
- `POST /api/v1/groups/{id}/leave/` — 退出小组
- `POST /api/v1/groups/{id}/members/{user_id}/manage/` — 管理成员
- `GET /api/v1/groups/{id}/posts/` — 群组动态

**Auth 模块：**
- `GET /api/v1/auth/me/` — 当前用户信息
- `PATCH /api/v1/auth/me/` — 更新个人信息
- `POST /api/v1/auth/logout/` — 退出登录
- `POST /api/v1/auth/change-password/` — 修改密码
- `POST /api/v1/auth/avatar/` — 上传头像
- `POST /api/v1/auth/cover/` — 上传封面

**其他模块：**
- 好友、聊天、通知、插件的所有端点均保持 `IsAuthenticated`

| 打开问卷/提问箱 | 个人主页插件卡片 | `/profile/{userId}/plugins/{plugin}` |
| 查看好友/聊天/通知/设置 | 导航栏链接 | 对应的功能路由 |

    def get(self, request):
        if request.user.is_authenticated:
            user = request.user
            friend_ids = Friendship.objects.filter(
                Q(from_user=user) | Q(to_user=user),
                status=Friendship.Status.ACCEPTED,
            ).values_list("from_user_id", "to_user_id")
            friend_id_set = set()
            for from_id, to_id in friend_ids:
                friend_id_set.add(from_id)
                friend_id_set.add(to_id)

                Q(visibility=Post.Visibility.PUBLIC)
                | Q(visibility=Post.Visibility.FRIENDS, author_id__in=friend_id_set)

    try:
        post = Post.objects.get(pk=pk)
    except Post.DoesNotExist:
        return api_response(
            code=40004, message="动态不存在", data=None,
            status=status.HTTP_404_NOT_FOUND,
        )

    if not request.user.is_authenticated:
        # 访客只能查看公开动态
        if post.visibility != Post.Visibility.PUBLIC:
            return api_response(
                code=40003, message="无权查看该动态", data=None,

        # 登录用户：现有权限检查逻辑
        if post.visibility == Post.Visibility.PRIVATE and post.author != request.user:
            return api_response(...)
        if post.visibility == Post.Visibility.FRIENDS and post.author != request.user:
            is_friend = Friendship.objects.filter(...).exists()
            if not is_friend:
                return api_response(...)

    return api_response(data=PostSerializer(post, context={"request": request}).data)

async function handleLogin() {
  if (!username.value || !password.value) {
    error.value = '请填写用户名和密码'
    return
  }
  loading.value = true
  error.value = ''
  try {

  } finally {
    loading.value = false
  }
}

## 十、注意事项

1. **匿名用户与 request.user**：Django 中匿名用户实例为 `AnonymousUser`，其 `is_authenticated` 属性为 `False`。所有 `request.user` 的引用需要确保兼容。

2. **PostSerializer 上下文**：`PostSerializer` 接受 `context={"request": request}`，序列化器中可能使用 `request.user` 判断是否点赞等。确认序列化器在匿名用户时也正常工作（返回 `is_liked: false`）。

3. **CSRF Token 与访客 GET 请求**：访客 GET 请求不需要 CSRF token，不会触发 CSRF 中间件报错。无需额外处理。

4. **Image/File 访问权限**：帖子中的图片和文件通过 Django 的 `serve()` 或存储 URL 访问，不受 DRF 权限控制。如果使用了 `private media` 机制（如 `X-Accel-Redirect`），需要确认访客是否可以访问公开帖子的图片。

5. **搜索引擎兼容**：目前是 SPA，搜索引擎看不到内容。如需 SEO，后续可考虑 SSR（Nuxt）或预渲染方案，不在本次范围内。

6. **性能考虑**：开放 GET 端点后，未认证请求不会走 `SessionAuthentication` 的 session 查询，但每次请求都会查询数据库。如果访客流量大增，可考虑加 Redis 缓存，不在本次范围内。
