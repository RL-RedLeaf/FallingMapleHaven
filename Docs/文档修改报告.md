## 3. Docs/03-API与WebSocket接口文档.md

### 3.1 全局修正

| 位置 | 文档旧内容 | 新内容 | 原因 |
|-----|-----------|-------|------|
| 错误码表 | 含 40002、40006 | 移除未使用的错误码，资源不存在统一 40004 | 代码仅定义 40001/40003/40004/40005 |
| 认证方式 | 同时列出 Session + Token | 保留两种说明 | 两种均可用（CSRF exempt 的 API 视图使用 Token） |

### 3.2 接口修正

| 接口 | 文档旧内容 | 修正后 | 原因 |
|-----|-----------|-------|------|
| `POST /api/v1/auth/me/` → 响应 | `"avatar": ...` | `"avatar_url": ...` | `UserSerializer` 实际字段名 |
| `GET/PUT /api/v1/users/{id}/profile/` → 响应 | 含 `"user": { ... }` 嵌套 | 平铺展开（`bio`, `website`, `avatar_url`, `cover_url`, `plugins` 等） | 实际返回无 `user` 包装 |
| `GET /api/v1/users/{id}/profile/` → 响应 | `plugins` 含完整数据 | `plugins` 为空列表 `[]` | `ProfileSerializer.get_plugins` 返回 `[]` |
| `GET/PUT /api/v1/visitors/` → 响应 | `"count": N` | `"total": N` | 实际 paginator 返回 `total` |
| `GET /api/v1/guestbook/{user_id}/` | 路径含 `{user_id}` | 改为 `GET /api/v1/guestbook/messages/?user_id=X` | 实际 ViewSet 使用查询参数 |
| `POST /api/v1/guestbook/{user_id}/` | 同上 | 改为 `POST /api/v1/guestbook/messages/` | 同上 |
| `DELETE /api/v1/guestbook/{id}/` | 同上 | 改为 `DELETE /api/v1/guestbook/messages/{id}/` | 同上 |
| `GET /api/v1/posts/{post_id}/` | OK | 无变化 | 确认正确 |
| `DELETE /api/v1/posts/{post_id}/` | OK 但缺少 `/delete/` | 改为 `DELETE /api/v1/posts/{post_id}/delete/` | URL 配置含 `/delete/` 后缀 |
| `POST /api/v1/posts/` | 请求体 `visibility` 缺默认可选 | 补充 `"visibility": "public"` 默认值 | 完善说明 |
| 评论列表/创建/删除 | 路径正确 | 无变化 | 确认正确 |
| `POST /api/v1/search/history/` | 响应不含 `id` | 补充 `"id"` 字段 | 实际返回含 pk |
| `POST /api/v1/notifications/mark_read/` | 响应不含 `"message"` | 补充 `"message": "已读"` | 实际返回值 |
| WebSocket 消息类型 | `friend_request`, `friend_accept` | `friend_request`, `friend_request_accepted` | 代码中 consumer 定义 |
