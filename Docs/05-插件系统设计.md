## 1. 设计目标

- **可插拔**：每个互动工具是一个独立 Django App，可独立启停
- **统一注册**：插件通过统一机制注册到系统
- **前后端协同**：后端注册 API 和模型，前端注册路由和组件
- **管理员可控**：管理后台可控制每个插件的启用/禁用
- **可扩展**：后期添加新插件不需要修改核心代码

## 2. 后端插件架构

### 2.1 插件基类

```python
# backend/apps/plugins/base.py

from abc import ABC, abstractmethod


class BasePlugin(ABC):
    """所有插件必须继承的基类"""

    # 插件元信息
    plugin_type: str = ""           # 唯一标识，如 "quiz"
    display_name: str = ""           # 显示名称，如 "默契问答"
    description: str = ""            # 描述
    icon: str = ""                   # 图标标识

    @abstractmethod
    def get_profile_data(self, user_id: int) -> dict:
        """获取在个人主页展示的数据"""
        pass

    def get_routes(self) -> list:
        """返回该插件的前端路由定义（非抽象方法，默认返回空列表）"""
        return []

    def get_admin_config(self) -> dict:
        """返回管理员配置页面需要的信息"""
        return {}
```

### 2.2 插件注册器

```python
# backend/apps/plugins/registry.py

class PluginRegistry:
    _plugins = {}

    @classmethod
    def register(cls, plugin_class):
        instance = plugin_class()
        cls._plugins[instance.plugin_type] = instance
        return plugin_class

    @classmethod
    def get_plugin(cls, plugin_type):
        return cls._plugins.get(plugin_type)

    @classmethod
    def get_active_plugins(cls):
        """获取已启用的插件列表（从数据库 Plugin 模型查询）"""
        from .models import Plugin
        active_types = Plugin.objects.filter(
            is_active=True
        ).values_list('type', flat=True)
        return {
            ptype: cls._plugins[ptype]
            for ptype in active_types
            if ptype in cls._plugins
        }

    @classmethod
    def get_all_plugins(cls):
        return dict(cls._plugins)

    @classmethod
    def get_profile_plugins(cls, user_id: int) -> list:
        """获取某个用户主页应该展示的插件数据"""
        result = []
        for plugin in cls.get_active_plugins().values():
            data = plugin.get_profile_data(user_id)
            if data:
                result.append(data)
        return result


# 装饰器，方便注册
def register_plugin(cls):
    PluginRegistry.register(cls)
    return cls
```

### 2.3 插件注册方式

```python
# backend/apps/plugins/quizzes/plugin.py

from ..base import BasePlugin
from ..registry import register_plugin


@register_plugin
class QuizPlugin(BasePlugin):
    plugin_type = "quiz"
    display_name = "默契问答"
    description = "和朋友互相出题测试默契度"
    icon = "help-circle"

    def get_profile_data(self, user_id):
        from .models import QuizResult
        from django.db.models import Q

        # 返回最近互动的几位好友的匹配度
        results = QuizResult.objects.filter(
            Q(user_a_id=user_id) | Q(user_b_id=user_id)
        ).order_by('-updated_at')[:5]

        return {
            "type": self.plugin_type,
            "name": self.display_name,
            "icon": self.icon,
            "description": self.description,
            "data": {
                "friend_count": results.count(),
                "top_friends": [
                    {
                        "friend_id": r.user_a_id if r.user_b_id == user_id else r.user_b_id,
                        "match_score": float(r.match_score),
                    }
                    for r in results
                ],
            },
            "route": f"/profile/{user_id}/plugins/quiz",
        }

    def get_routes(self):
        return [
            {
                "path": "/profile/:userId/plugins/quiz",
                "name": "QuizPage",
                "component": "QuizPage.vue",
            }
        ]
```

### 2.4 插件模型 (数据库)

```python
# backend/apps/plugins/models.py

from django.db import models


class Plugin(models.Model):
    name = models.CharField(max_length=100)           # 显示名称
    type = models.CharField(max_length=50, unique=True)  # 类型标识
    is_active = models.BooleanField(default=True)     # 是否启用
    config = models.JSONField(null=True, blank=True)  # 插件配置
    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        db_table = 'plugins_plugin'
```

## 3. 前端插件架构

### 3.1 插件注册机制

```javascript
// frontend/src/plugins/registry.js

class PluginRegistry {
  constructor() {
    this.plugins = new Map()
  }

  register(pluginConfig) {
    // pluginConfig: { type, name, icon, component, route, profileData }
    this.plugins.set(pluginConfig.type, pluginConfig)
  }

  getPlugin(type) {
    return this.plugins.get(type)
  }

  getActivePlugins() {
    // 从后端获取启用的插件列表
    return Array.from(this.plugins.values())
  }

  getRoutes() {
    // 收集所有插件的路由定义
    const routes = []
    for (const plugin of this.plugins.values()) {
      if (plugin.route) {
        routes.push(plugin.route)
      }
    }
    return routes
  }
}

export const pluginRegistry = new PluginRegistry()
```

### 3.2 插件注册示例

```javascript
// frontend/src/plugins/quiz/index.js

import { pluginRegistry } from '../registry'

pluginRegistry.register({
  type: 'quiz',
  name: '默契问答',
  icon: 'help-circle',
  route: { path: '/profile/:userId/plugins/quiz', name: 'QuizPage' },
})
```

> 当前实现中不传入 `component` 和 `cardComponent`，前端路由在 `router/index.js` 中硬编码。由于当前版本仅交付首批两个插件，这不作为当前功能缺口；若后续要支持“新增插件不改路由”，再启用 `pluginRegistry.getRoutes()` 动态加载。

### 3.3 动态路由加载

> 低优先级技术债：插件路由目前在 `router/index.js` 中硬编码，未使用 `pluginRegistry.getRoutes()` 动态加载。当前版本首批插件数量固定，可暂不处理。

**预期的实现方式：**
```javascript
// frontend/src/router/index.js

import { createRouter, createWebHistory } from 'vue-router'
import { pluginRegistry } from '@/plugins/registry'

const staticRoutes = [
  // ... 静态路由定义
]

// 动态添加插件路由
const pluginRoutes = pluginRegistry.getRoutes()
const router = createRouter({
  history: createWebHistory(),
  routes: [...staticRoutes, ...pluginRoutes],
})

export default router
```

### 4.3 新插件添加步骤

如果需要添加一个新插件（例如"投票问卷"），需要完成：

**后端:**
1. 在 `backend/apps/plugins/` 下创建新目录 `polls/`
2. 创建 `models.py` (Poll, PollOption, PollVote)
3. 创建 `serializers.py` (序列化器)
4. 创建 `views.py` (API 视图)
5. 创建 `urls.py` (路由)
6. 创建 `plugin.py` (继承 BasePlugin，用 @register_plugin 装饰)
7. 运行 `makemigrations` + `migrate`
8. 在 `Plugin` 表中添加一条记录 `{type: "poll", name: "投票问卷", is_active: true}`

**前端:**
1. 在 `frontend/src/plugins/` 下创建 `polls/` 目录
2. 创建 `PollsPage.vue`
3. 创建 `index.js` (调用 `pluginRegistry.register`)
4. 在 `main.js` 中 `import '@/plugins/polls'`

## 5. 首批插件详设

### 5.1 默契问答

**数据模型:**
```
QuizQuestion
├── id: PK
├── creator: FK→User          # 出题人
├── target_user: FK→User      # 答题人
├── question: TEXT             # 题目
├── correct_answer: VARCHAR    # 正确答案
├── created_at: DATETIME

QuizAnswer
├── id: PK
├── question: FK→QuizQuestion (UNIQUE)
├── answerer: FK→User
├── answer: VARCHAR
├── is_correct: BOOLEAN        # 由系统比对填充
├── created_at: DATETIME

QuizResult
├── id: PK
├── user_a: FK→User
├── user_b: FK→User
├── total_questions: INT
├── correct_count: INT
├── match_score: DECIMAL(5,2)  # correct_count/total_questions * 100
├── updated_at: DATETIME
└── UNIQUE(user_a, user_b) 且 user_a < user_b
```

**核心流程:**
1. 用户A 在 B 的主页 → 点击默契问答 → 出题
2. 系统创建 QuizQuestion，记录 creator=A, target_user=B
3. 用户B 收到通知 → 进入答题页面
4. B 提交答案 → 系统比对 → 更新 QuizResult
5. 用户A 主页插件卡片显示与 B 的匹配度

**API:**
```
POST   /api/v1/plugins/quiz/questions/           # 出题
GET    /api/v1/plugins/quiz/questions/received/   # 我收到的题
POST   /api/v1/plugins/quiz/questions/{id}/answer/  # 答题
GET    /api/v1/plugins/quiz/result/{friend_id}/   # 匹配度
DELETE /api/v1/plugins/quiz/questions/{id}/       # 删除题目
```

### 5.2 匿名提问箱

**数据模型:**
```
AnonymousQuestion
├── id: PK
├── recipient: FK→User         # 被提问者
├── sender: FK→User            # 提问者(普通用户不可见)
├── content: TEXT               # 提问内容
├── answer: TEXT                # 回答内容(null 表示未回答)
├── is_answered: BOOLEAN
├── is_public: BOOLEAN          # 是否公开展示
├── created_at: DATETIME
├── answered_at: DATETIME
```

**核心流程:**
1. 用户C 访问 A 的主页 → 点击匿名提问箱 → 输入问题
2. 系统创建 AnonymousQuestion，recipient=A，sender=C
3. A 在提问箱页面看到问题列表（不显示 C 的身份）
4. A 可选择回答（公开）或不回答
5. 已回答的问答展示在 A 的提问箱页面
6. 管理员后台可查看 sender 真实身份

**API:**
```
POST   /api/v1/plugins/question-box/                            # 提问
GET    /api/v1/plugins/question-box/inbox/                       # 我收到的提问
POST   /api/v1/plugins/question-box/{id}/answer/                 # 回答
DELETE /api/v1/plugins/question-box/{id}/delete/                 # 删除
```

**管理员可见真实提问者:**
```python
# 普通用户序列化
class AnonymousQuestionReadSerializer(serializers.ModelSerializer):
    class Meta:
        model = AnonymousQuestion
        exclude = ['sender']  # 对普通用户隐藏

# 管理员序列化
class AdminAnonymousQuestionSerializer(serializers.ModelSerializer):
    real_sender = serializers.SerializerMethodField()

    def get_real_sender(self, obj):
        return {
            "user_id": obj.sender.id,
            "nickname": obj.sender.nickname,
        }

    class Meta:
        model = AnonymousQuestion
        fields = '__all__'
```

```
GET /api/v1/admin/plugins/
```

**响应:**
```json
{
    "code": 0,
    "data": [
        {
            "id": 1,
            "name": "默契问答",
            "type": "quiz",
            "is_active": true,
            "config": {}
        },
        {
            "id": 2,
            "name": "匿名提问箱",
            "type": "question_box",
            "is_active": true,
            "config": {}
        }
    ]
}
```

### 6.2 启停插件

```
PATCH /api/v1/admin/plugins/{id}/
```

**请求体:**
```json
{
    "is_active": false
}
```

> 当前后端主页插件入口通过 `PluginRegistry.get_active_plugins()` 过滤，禁用后会从用户主页隐藏。默契问答与匿名提问箱 API 也已通过 `ensure_plugin_active()` 接入启用状态检查，禁用后直接请求 API 返回 404。
