# 用户体验探索与需求符合度报告

> 日期：2026-06-28  
> 入口：http://localhost:5173/  
> 目的：用多名模拟用户从网页入口自由学习和探索，验证产品是否能达到需求预期，并记录实际体验、阻塞点和与代码审计结论互相印证的问题。  
> 说明：本报告不提出代码修复方案，只做项目经理视角的需求符合度、体验风险和问题分组。具体修复工作交由程序员处理。

---

## 一、总体结论

项目已经具备一个社交主页产品的基本骨架：注册、登录、首页动态、发帖、点赞、评论、通知、个人主页、设置页、好友页、聊天页、插件入口和管理后台均已在前端或后端出现。实际探索中，普通社交链路的部分闭环已经跑通，例如注册登录、发布动态、另一用户查看动态、点赞评论、收到通知、后台登录和后台数据读取。

但当前还不能按 PRD 直接验收为“核心体验可用”。主要原因不是单个页面缺失，而是多个关键闭环存在入口弱、路由/API 不一致、认证初始化不稳定、移动端导航错误、设置保存失败、聊天未能建立会话、插件入口不清晰等问题。部分问题已被代码审计直接佐证，属于确定缺陷；部分问题是用户探索未完成或被工具限制，应标为“未验证/高风险”，不能简单等同于功能完全损坏。

优先级判断如下：

| 优先级 | 判断 | 代表问题 |
|---|---|---|
| P0 | 阻断核心闭环或会导致功能必然失败 | 匿名提问前后端路径不一致、小组创建路径不一致、聊天 WebSocket 事件不一致、刷新/深链进入受保护页可能误跳登录 |
| P1 | 明显影响体验、验收或数据正确性 | 移动端“我的”指向登录页、设置保存 CSRF 失败、空状态缺少下一步、注册后落点与 PRD 不一致、群组动态隔离逻辑风险 |
| P2 | 影响完整度、运营效率或边界体验 | 404 空白、后台移动端入口缺失、插件入口弱、日志/设置有效性未验证、按钮触控高度略低 |

## 四、未完成或失败的关键闭环

### 4.1 好友闭环

| 需求预期 | 探索结果 | 判断 |
|---|---|---|
| 搜索用户、发送好友申请、接收/同意好友申请 | 社交用户探索时尝试进入好友流程，但在“搜索用户”附近被卡住，未完整完成申请和接受 | 未验证完成，高风险 |

这不能直接证明好友功能完全不可用，但从用户体验角度看，好友入口、搜索入口、下一步引导不够稳定或不够明显。由于聊天依赖好友或会话关系，好友闭环未跑通会连带影响聊天验收。

### 4.2 聊天闭环

| 需求预期 | 探索结果 | 判断 |
|---|---|---|
| 建立好友后可聊天，消息实时到达 | 因好友/会话未建立，聊天页仅显示“暂无聊天”，未能完成真实收发消息 | 未验证完成，高风险 |

代码审计另有确定问题：后端 WebSocket 发送 `type: "new_message"`，前端 store 期待 `type === "message"` 且读取 `data.message`。这会导致即便建立会话，实时消息处理也存在较大失败风险。

### 4.3 插件闭环

| 插件 | 探索结果 | 代码佐证 | 判断 |
|---|---|---|---|
| 匿名提问箱 | 用户侧未找到稳定可用的完整路径；直接 `/plugins`、`/quiz`、`/questions` 有路由不匹配警告 | 前端请求 `/plugins/question-box/questions/`，后端实际是 `/plugins/question-box/` | 确定 P0 风险 |
| 默契问答/Quiz | 插件模块存在，但自由探索中入口不明显，完整答题/匹配未验证 | 代码存在 quiz 模块，但业务校验不足 | 未验证完成 |
| 主页插件卡片 | 主页可见插件卡片机制 | `PluginRegistry.get_profile_plugins()` 存在 | 基础结构成立 |

插件是项目的差异化能力，但当前入口和路径一致性不足，不能按“插件系统可完整验收”处理。

### 4.4 设置与资料编辑

| 需求预期 | 探索结果 | 判断 |
|---|---|---|
| 用户能编辑个人资料、上传头像、保存设置 | 直接 PATCH 设置和头像上传出现 `CSRF Failed: CSRF token missing`，设置页对错误捕获较静默 | P1 体验与功能风险 |

这类问题会严重影响用户对“个人主页可定制”的感知，也会影响 PRD 中关于个人主页装修、头像、封面、隐私设置的验收。

### 4.5 错误页与空状态

| 场景 | 实际表现 | 判断 |
|---|---|---|
| 未匹配路由 | 页面区域空白，无清晰 404 | P2 |
| 聊天空状态 | 只显示“暂无聊天”，缺少添加好友/发起聊天等下一步动作 | P1 |
| 好友/插件入口 | 自由探索时不够明确 | P1 |

## 五、他们是否被卡住：逐视角检查

### 5.1 新用户探索

未整体卡死，主路径完成度较高。该视角完成了从未登录访问、注册、登录、浏览首页、发动态、访问个人主页、进入设置/通知/好友/聊天等页面的学习过程。

| 未知路由空白 | 体验缺失 | 学习和试错成本高 |

### 5.2 活跃社交用户探索

没有卡在注册、登录、发帖、点赞、评论、通知这些基础社交流程上，这部分已形成可用闭环。

主要卡点：

| 卡点 | 类型 | 说明 |
|---|---|---|
| 好友搜索/申请流程未完成 | 用户路径阻塞 | 探索中停在“搜索用户”附近，未能稳定完成加好友 |

### 5.3 管理员探索

没有卡在权限和后台读取能力上。管理员能登录，能看到后台入口，后台主要读取接口均返回成功；普通用户访问后台接口返回 403。

主要卡点：

| 卡点 | 类型 | 说明 |
|---|---|---|
| 直接刷新 `/admin/*` 可能误跳 `/login` | 产品缺陷 | 与普通受保护页相同的认证初始化竞态 |
| 移动端后台入口不可见或难以访问 | 体验/适配风险 | 小屏管理场景未覆盖 |
| 匿名问题列表为空 | 数据条件不足 | 未验证真实发送者展示和处理动作 |
| 插件启停、删除、审核等写操作未执行 | 测试边界 | 为避免污染数据，本轮只验证读和权限 |
| 日志/设置为空或缺少可观察效果 | 验证不足 | 审计和配置闭环仍需专项验收 |

结论：后台读能力基本成立，后台管理闭环未完全验收。

## 六、跨视角共同问题分组

### 6.1 认证和路由稳定性

| 问题 | 影响 | 证据 |
|---|---|---|
| 刷新或直接进入受保护页面时可能误跳 `/login` | 用户以为登录失效，后台和设置页均受影响 | 多个视角观察到 |

| 未知路由空白 | 用户试错时没有解释和返回路径 | 新用户探索观察到 |

| 问题 | 影响 | 证据 |

| 匿名提问箱前端路径与后端路径不一致 | 匿名提问相关功能可能直接失败 | 代码审计确定 |
| 小组创建前端路径与后端路径不一致 | 小组创建可能直接失败 | 代码审计确定 |
| WebSocket 事件类型不一致 | 聊天实时消息可能无法被前端处理 | 代码审计确定 |
| 首页发帖字段 `tag`/`topic_tag` 不一致 | 话题标签可能丢失或失败 | 代码审计确定 |

### 6.3 核心社交闭环

## 十一、简短结论

当前项目不是空壳，基础社交动态已经能跑出真实互动闭环；后台也不是纯页面，权限和读取接口基本成立。但产品距离 PRD 的“个人主页 + 社交 + 插件 + 后台管理”完整体验仍有明显差距。

本轮模拟用户主要卡在三个层面：

1. 认证和导航稳定性：刷新、深链、移动端“我的”会让用户误以为自己未登录或找不到个人中心。
2. 深层社交关系链：好友流程没有自然跑通，聊天因此无法验收，且代码层还有实时消息协议不一致。
3. 插件和小组闭环：入口不清晰，且匿名提问、小组创建等已被代码确认存在路径错误。

建议后续不要按“页面是否存在”验收，而按本报告第九章的 A-G 闭环逐组验收。程序员修复 P0 后，应优先重测“好友 + 聊天”“匿名提问”“小组创建”“设置保存”“深链刷新保持登录态”这五条路径。

### 12.1 当前代码仍可佐证的问题

| 编号 | 结论 | 代码证据 | 影响 |

| R1 | 受保护路由刷新或深链进入仍可能误跳登录 | `frontend/src/router/index.js` 的 `beforeEach` 直接检查 `authStore.isAuthenticated`；`frontend/src/App.vue` 在 `onMounted()` 后才调用 `authStore.fetchMe()` | 已有 session 的用户直接访问 `/settings`、`/admin`、`/chat` 等页面时，路由守卫可能早于登录态恢复执行，误跳 `/login` |

| L5 | 通用 `/api/v1/upload/` 路由已注册但当前前端未使用 | 从全局 URL 中移除 `api/v1/upload/` 路由，避免暴露当前版本未使用的通用上传入口；保留 `apps.upload` 代码，后续需要时可显式接回 | `backend/config/urls.py`, `backend/apps/upload/views.py` |
